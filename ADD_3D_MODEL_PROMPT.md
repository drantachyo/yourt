# 🎨 Промпт для добавления вращаемой 3D модели на сайт

Используйте этот промпт когда нужно добавить новую интерактивную 3D модель на сайт наподобие юрты.

---

## 📋 Шаблон промпта

```
Добавь на страницу новый интерактивный 3D модуль, подобно существующей юрте.

**Параметры:**
- Название компонента: [NAME] (например: "Казахский Дом", "Кибитка", "Боевой Конь")
- Файл GLB модели: [FILE.glb] (название файла, который будет в папке проекта)
- Позиция на странице: [LOCATION] (например: "после юрты", "новая секция")
- Компоненты модели, которые нужно выбирать: [COMPONENTS] (список из 3-4 элементов)

**Описания компонентов (для меню выбора):**
- Компонент 1: [описание на русском/казахском/английском]
- Компонент 2: [описание]
- ...

**Аудио описания** (MP3 файлы):
- [component_ru.mp3], [component_kz.mp3], [component_en.mp3]
- (или указать что аудио не требуется)

**Стиль/тема:**
- Цветовая схема: [какие цвета использовать]
- Размер 3D контейнера: [по умолчанию 72vh]
- Особенности: [любые специальные требования]
```

---

## 🔧 Как это работает (техническая часть для подсказки)

1. **GLB модель** загружается в Three.js сцену через GLTFLoader
2. **Автоматическое масштабирование** и центрирование модели
3. **OrbitControls** - пользователь может вращать, зумить через мышь
4. **Выбор компонентов** через боковое меню - выбранный компонент светится, остальные становятся полупрозрачными
5. **i18n система** - поддержка RU/KZ/EN через `[data-i18n]` атрибуты
6. **Аудио плеер** - MP3 описания проигрываются при выборе компонента
7. **Информационная карточка** - появляется справа с описанием выбранного компонента

---

## 📁 Файлы которые нужны

1. **GLB модель** (основной файл): `model_name.glb`
2. **Аудио файлы** (опционально):
   - `component1_ru.mp3`, `component1_kz.mp3`, `component1_en.mp3`
   - `component2_ru.mp3`, `component2_kz.mp3`, `component2_en.mp3`
   - ...

## 🎯 Ключевые шаги при добавлении модели

1. Положить GLB файл в корень папки проекта
2. Указать файл в GLB_PARTS массиве в коде JavaScript
3. Добавить i18n ключи для названий и описаний компонентов
4. Добавить кнопки меню для выбора компонентов
5. Связать аудио файлы через AUDIO_FILE_MAP
6. Сделать новую секцию HTML с контейнером #[id]-canvas-container

---

## 💡 Полезные советы

- **GLB модель должна быть в формате glTF** (GLB это бинарный формат glTF)
- **Размер файла**: желательно < 5MB для быстрой загрузки
- **Полигоны**: оптимально 50K-200K полигонов
- **Материалы**: сохранять стандартные материалы, избегать сложных шейдеров
- **Текстуры**: встраивать прямо в GLB (не внешние файлы)
- **Компонентность**: разные части модели должны быть отдельными объектами/группами в GLB для raycast выбора
